# Inventory & Order Management System (Test Assignment)

Веб-додаток для моніторингу замовлень, товарів та активних користувацьких сесій у реальному часі. Проєкт побудований на базі архітектури SPA з інтеграцією WebSocket та повною контейнеризацією середовища.

## 🚀 Технологічний стек

- **Frontend:** React 19, TypeScript, Vite 8, Redux Toolkit, Framer Motion, Bootstrap 5.
- **Backend:** Node.js, Express, Socket.io (трансляція годинника та активних сесій).
- **DevOps & Deployment:** Docker, Docker Compose, Nginx.

---

## 🛠 Інструкція із запуску

### Варіант 1: Запуск через Docker (Рекомендовано)

Переконайся, що у тебе встановлено та запущено Docker Desktop (з увімкненою апаратною віртуалізацією в BIOS).

1. Відкрий термінал у кореневій папці проєкту.
2. Виконай команду для автоматичного збирання та підняття контейнерів:

```bash

# Для нових версій Docker (v2+)

docker compose up --build

# Для старіших версій Docker

docker-compose up --build
```

3. Після успішного завершення збірки додаток буде доступний за адресою:
   - **Frontend:** http://localhost:3000
   - **Backend:** http://localhost:3001

_Щоб зупинити контейнери, натисни Ctrl + C у вікні терміналу._

---

### Варіант 2: Локальний запуск у режимі розробника (Без Docker)

Якщо ти віддаєш перевагу ручному запуску без контейнерів, у корені монорепозиторію вже налаштовано пакет `concurrently` для одночасного старту обох частин додатка.

1. У головному корені проєкту (там, де лежить загальний package.json) встанови залежності:

```bash
   npm install
```

2. Запусти фронтенд та бекенд разом однією командою:

```bash
   npm run dev
```

_Ця команда автоматично запустить Express-сервер на порту 3001 та Vite-клієнт у розробницькому режимі в одному вікні терміналу, розділивши логи за кольорами._
